한국어 인터페이스의 웹 서비스 UI를 설계해줘.

프로젝트명: Handover HQ
한 줄 소개: 명세서만 받아도 참가, 팀 구성, 제출, 결과 확인까지 이어지는 해커톤 운영 허브

목표:
- 해커톤 운영 웹 서비스를 현대적인 SaaS 스타일로 설계
- 필수 명세를 빠짐없이 담되, 실제 서비스처럼 사용성이 좋아야 함
- 초보 개발자도 길을 잃지 않도록 “다음 행동”이 분명해야 함
- 반응형 웹 기준으로 설계

디자인 방향:
- 전체 톤은 깔끔하고 신뢰감 있는 생산성 SaaS
- 흰 배경 중심, 진한 네이비 포인트, 따뜻한 옐로 포인트를 보조색으로 사용
- 느낌은 “메모/도면/인수인계 문서”에서 영감을 받되 너무 장난스럽지 않게
- 카드 기반 레이아웃, 명확한 정보 위계, 넉넉한 여백, 높은 가독성
- 한국어 타이포 중심
- 데스크톱 1440px 기준, 모바일 390px 기준 반응형도 함께 고려
- 접근성 좋은 대비, 버튼 상태, 폼 상태를 분명히 표현

반드시 설계할 페이지:
1. 메인 페이지 (/)
2. 해커톤 목록 (/hackathons)
3. 해커톤 상세 (/hackathons/:slug)
4. 팀 모집 (/camp)
5. 랭킹 (/rankings)

공통 UI 요구사항:
- 상단 헤더: 로고, 해커톤, 팀 찾기, 랭킹, CTA 버튼
- 공통 푸터
- 로딩 상태, 빈 상태, 오류 상태까지 포함
- 반응형 구조
- 재사용 가능한 컴포넌트 시스템 포함
- 버튼, 탭, 배지, 카드, 표, 필터 바, 입력 폼, 토스트, 모달까지 컴포넌트화

페이지별 상세 요구사항:

1) 메인 페이지
- 큰 히어로 영역
- 메인 카피: “명세서만 받아도 구현부터 제출까지”
- 큰 CTA 3개:
  - 해커톤 보러가기
  - 팀 찾기
  - 랭킹 보기
- 진행 중 해커톤 수, 모집 중 팀 수, 가까운 마감 등 요약 통계 카드
- 진행 중 해커톤 미리보기 카드 2~3개
- 빠른 시작 영역: “지금 무엇을 하면 되는지” 보여주는 가이드 카드

2) 해커톤 목록
- 상단 제목, 검색/필터/정렬 바
- 상태 필터: 예정 / 진행 중 / 종료
- 태그 필터
- 정렬: 최신순 / 마감 임박순
- 해커톤 카드 리스트
- 각 카드에 제목, 상태, 태그, 제출 마감일, 종료일, 참가 규모 요약 표시
- 카드 클릭 시 상세 페이지 이동
- 로딩 / 데이터 없음 / 오류 상태 UI 포함

3) 해커톤 상세
- 이 페이지가 서비스의 핵심 허브
- 상단에 해커톤 제목, 태그, 상태, D-day, CTA 배치
- 우측 또는 상단 요약 카드:
  - 팀 정책
  - 현재 해야 할 다음 행동
  - 제출 상태
  - 마감 정보
- 탭 구조 또는 섹션 네비게이션:
  - 개요
  - 안내
  - 평가
  - 상금
  - 일정
  - 팀
  - 제출
  - 리더보드
- 각 섹션은 정보 카드, 표, 타임라인 등으로 보기 좋게 배치
- 정보가 없는 섹션은 “추후 공개” 또는 “정보 없음” 카드로 자연스럽게 처리
- 일정은 타임라인 컴포넌트로 디자인
- 제출 섹션은 단계형 제출 UI로 설계:
  - 기획서
  - 웹 페이지 배포 URL
  - GitHub 저장소 링크
  - 솔루션 PDF
- 제출 섹션에는 다음 요소 포함:
  - 자동 임시저장 표시
  - 체크리스트
  - URL 검증 상태
  - 제출 버튼
  - 최종 제출 전 확인 모달
- 리더보드는 표 형태 + 상위 팀 강조 카드
- 리더보드 표에는 순위, 팀명, 점수, 제출 시각, 참가자 점수, 심사위원 점수, 산출물 링크 표시

4) 팀 모집 페이지
- 해커톤별 필터
- 모집 중만 보기 토글
- 역할 필터(프론트엔드, 백엔드, 디자이너, PM 등)
- 팀 카드 리스트
- 팀 카드에 팀명, 연결된 해커톤, 현재 인원, 모집 상태, 찾는 역할, 소개, 연락 링크 표시
- “팀 만들기” 버튼과 생성 폼 모달 또는 별도 영역
- 팀 생성/수정 폼에는 팀명, 소개, 모집 여부, 찾는 역할, 연락 링크 입력
- UX 확장으로 “팀 적합도” 시각화 포함:
  - 사용자의 관심 역할과 팀 모집 역할이 얼마나 맞는지 배지나 점수로 표현

5) 랭킹 페이지
- 전체 랭킹 테이블
- 기간 필터: 최근 7일 / 30일 / 전체
- rank / 팀명 / points 중심 구조
- 상단에 랭킹 설명 카드
- 상위 3팀은 강조 카드로 별도 노출
- 상세 페이지 또는 제출물로 바로 이동 가능한 인터랙션 포함

꼭 반영할 UX 포인트:
- “Next Action” 컴포넌트:
  사용자 상태에 따라 다음 행동이 달라짐
  예: 팀이 없으면 팀 찾기/팀 만들기, 제출 전이면 제출하기, 제출 후면 리더보드 보기
- “Team Fit” 컴포넌트:
  관심 역할과 팀 모집 역할 매칭 표시
- “Submit Guard” 컴포넌트:
  자동 저장, 검증 상태, 체크리스트, 마감 카운트다운, 최종 확인 모달
- 심사자가 빠르게 이해할 수 있도록 핵심 정보는 첫 화면부터 드러나야 함

컴포넌트 시스템 요구사항:
- Primary / Secondary / Ghost 버튼
- Status badge (예정, 진행 중, 종료, 모집 중, 제출 완료 등)
- Filter chip
- Stat card
- Hackathon card
- Team card
- Ranking row / leaderboard row
- Section tab
- Timeline item
- Empty state / Error state / Loading skeleton
- Form field / URL validation state / helper text
- Toast / Confirm modal

산출물 형태:
- 화면만 그리는 것이 아니라 실제 서비스처럼 설계된 고충실도 UI
- 각 페이지는 일관된 디자인 시스템을 공유
- 데스크톱 우선으로 설계하고 모바일 축소 구조도 자연스럽게 제안
- 깔끔하고 구현 가능한 현실적인 UI로 만들어줘
- 너무 과한 장식보다 정보 구조와 사용성을 우선해줘
