# 작업 로그

## 2026-03-24

### 1. 설계 분석
- 전달받은 DOCX 설계 문서와 현재 구현 코드를 대조했다.
- 현재 코드는 피그마 기반 정적 화면 중심이며, 실제 백엔드/인증/권한/제출 저장은 미구현 상태로 판단했다.

### 2. 에이전트 검토
- IT 서비스 기획 에이전트로 기능 체크리스트와 Supabase 도메인 모델 초안을 정리했다.
- 디자인 에이전트로 현재 톤을 유지한 상태에서 필요한 공통 패턴과 접근성 리스크를 점검했다.
- 코딩/보안 에이전트로 Auth, RLS, 외부 링크 처리, Vercel SPA 라우팅 리스크를 점검했다.

### 3. 이번 작업 목표
- 실서비스 전환 기획서를 저장한다.
- Supabase Auth와 공통 데이터 계층을 추가한다.
- 홈, 해커톤 목록, 상세, 팀 모집, 랭킹을 실제 데이터 구조에 맞게 전환한다.
- 제출 초안 저장과 최종 제출 흐름을 구현한다.
- 작업 단위별로 커밋한다.

### 4. 1차 구현 완료
- `feat: stabilize app state and hackathon workflows` 커밋으로 공통 `AppProvider`와 `useAppContext`를 추가했다.
- 홈, 해커톤 목록, 상세, 팀 모집, 랭킹을 Supabase/시드 공통 데이터 계층 기준으로 다시 연결했다.
- 상세 페이지에 `Submit Guard`, 자동 저장, 초안 저장, 최종 제출 확인 모달, 버전 이력 표시를 넣었다.
- 헤더를 새 인증 상태 구조에 맞게 정리했고, 디자인 톤은 `#0064FF` 중심으로 통일했다.
- 외부 링크 보안 처리와 URL 검증을 강화했다.
- `npm run typecheck`, `npm run build`를 확인했다.

### 5. 다음 작업
