## 2026-03-27

### 1. 인증/복귀 흐름 보강
- 로그인 전에 입력한 이름, 관심 역할, 복귀 URL을 pending profile로 저장하도록 정리했다.
- 매직링크와 GitHub 로그인 후에는 Auth metadata와 `profiles`를 함께 동기화하도록 보강했다.
- 해커톤 상세 탭을 URL hash 기반으로 바꿔 `#submit`, `#leaderboard` deep link와 로그인 복귀 흐름이 유지되도록 정리했다.

### 2. 제출 PDF 업로드와 권한 정리
- 해커톤 상세 제출 탭에 Supabase Storage 기반 PDF 업로드를 추가했다.
- `solutionPdfPath`를 제출/버전 모델에 추가하고, 업로드한 파일은 signed URL로 다시 읽도록 데이터 계층을 확장했다.
- 제출 저장과 최종 제출은 팀장만 진행할 수 있게 프론트 권한 체크를 넣었고, 팀원은 읽기 전용으로 보이도록 정리했다.
- Supabase 스키마에 Storage bucket, `solution_pdf_path` 컬럼, 관련 RLS 정책을 추가했다.

### 3. 운영 자동화와 배포 준비
- `scripts/seed-supabase.mjs`를 추가해 Auth 사용자, 프로필, 해커톤, 팀, 제출, 리더보드 시드를 한 번에 넣을 수 있게 했다.
- `package.json`에 `npm run seed:supabase` 스크립트를 추가했다.
- 라우트 lazy loading과 Vite manual chunk 설정을 넣어 번들 분할을 더 개선했다.
- `.env.example`, README, 구현 계획 문서를 현재 구조에 맞게 갱신했다.

### 4. 검증
- `npm run typecheck`
- `npm run build`

### 5. 실제 Supabase 연결 점검
- `.env`에 설정된 Supabase URL, anon key, service role key가 실제 값인지 확인했다.
- Storage bucket `hackathon-submissions`를 원격 프로젝트에 생성했다.
- `npm run seed:supabase`로 실제 프로젝트에 시드 데이터를 반영했고, 해커톤/팀/제출/리더보드 데이터가 생성되는 것을 확인했다.
- 공개 팀 조회 중 `team_members` 정책의 자기참조로 인한 RLS 무한 재귀를 발견했다.
