### 2. 제출 PDF 업로드와 권한 정리
- 해커톤 상세 제출 탭에 Supabase Storage 기반 PDF 업로드를 추가했다.
- `solutionPdfPath`를 제출/버전 모델에 추가하고, 업로드한 파일은 signed URL로 다시 읽도록 데이터 계층을 확장했다.
- 제출 저장과 최종 제출은 팀장만 진행할 수 있게 프론트 권한 체크를 넣었고, 팀원은 읽기 전용으로 보이도록 정리했다.
- Supabase 스키마에 Storage bucket, `solution_pdf_path` 컬럼, 관련 RLS 정책을 추가했다.

### 3. 운영 자동화와 배포 준비
- `scripts/seed-supabase.mjs`를 추가해 Auth 사용자, 프로필, 해커톤, 팀, 제출, 리더보드 시드를 한 번에 넣을 수 있게 했다.
- `package.json`에 `npm run seed:supabase` 스크립트를 추가했다.
- 라우트 lazy loading과 Vite manual chunk 설정을 넣어 번들 분할을 더 개선했다.
- `.env.example`, README, 구현 계획 문서를 현재 구조에 맞게 갱신했다.

### 4. 검증

- `npm run build`

## 2026-03-30

### 1. 로그인 UI 정리
- 로그인 후 헤더에 보이던 큰 사용자 카드와 `AUTH` 배지를 제거했다.
- 헤더는 `아바타 + 이름 + caret`만 보이는 일반적인 프로필 트리거 형태로 바꿨다.
- 이름을 클릭하면 드롭다운에서 이메일, 역할, 데모 여부, 로그아웃 버튼을 확인할 수 있게 정리했다.
- 바깥 영역 클릭과 `Escape` 입력 시 드롭다운이 닫히도록 처리했다.

### 2. 모바일 로그인 영역 정리
- 모바일 메뉴에서도 로그인 영역이 과하게 카드처럼 보이지 않도록 밀도를 낮췄다.
- 로그아웃 버튼은 큰 CTA 대신 작은 보조 액션처럼 보이도록 바꿨다.

### 3. 해커톤 정렬 UI 개편
- `마감일순/최신순`이 번갈아 바뀌는 토글 대신 실제 선택 메뉴가 열리는 드롭다운으로 바꿨다.
- 바깥 영역 클릭과 `Escape` 입력 시 메뉴가 닫히도록 처리했다.

### 4. 팀 공간 중심 UI 개편
- 팀 모집과 해커톤 상세의 팀 섹션에서 `연락하기` 버튼과 외부 메신저 링크 노출을 제거했다.
- 팀장 정보, 팀 공간 설명, 진행 현황, 최근 공유, 다음 체크포인트를 카드 안에서 바로 볼 수 있게 개편했다.
- 팀 생성/수정 폼도 외부 연락 링크 입력 대신 팀 소개와 진행상황을 자연스럽게 적도록 안내 문구를 바꿨다.
- 관련 기획을 [팀-공간-전환-계획.md](/Users/jiu/Desktop/Projects/dacon_hack/docs/팀-공간-전환-계획.md)에 정리했다.
