# 팀 참여 요청 기획 초안

## 1. 이번 라운드 최소 기능
- 팀 카드와 해커톤 상세 팀 섹션에 `지원하기` 버튼 추가
- 비로그인 사용자는 로그인 후 지원 흐름으로 복귀
- 지원자는 간단한 자기소개와 맡고 싶은 역할을 적어 요청 전송
- 팀장은 `대기 중 요청` 목록에서 요청을 확인하고 `승인/거절` 처리
- 승인 시 팀 멤버에 반영되고, 팀 정원이 가득 차면 자동으로 모집 종료 처리
- 지원자는 자신의 요청 상태를 `대기/승인/거절/취소`로 확인 가능

## 2. 데이터 모델 최소안

### `team_join_requests`
- `id`
- `team_id`
- `applicant_id`
- `requested_role`
- `intro_message`
- `status`
- `created_at`
- `updated_at`
- `reviewed_at`
- `reviewed_by`
- `decision_note`

### 상태값
- `pending`
- `accepted`
- `rejected`
- `cancelled`

### 최소 제약
- 같은 사용자는 같은 팀에 `pending` 상태 요청을 하나만 가질 수 있음
- 이미 팀 멤버인 사용자는 새 요청 생성 불가
- 모집 종료 팀에는 새 요청 생성 불가

## 3. 상태 흐름

### 지원자 기준
1. 팀 카드에서 `지원하기` 클릭
2. 역할 + 한 줄 소개 입력 후 요청 생성
3. 상태는 `대기 중`
4. 팀장 판단 후 `승인` 또는 `거절`
5. 승인되면 팀 멤버로 전환, 거절되면 상태만 기록
6. 대기 중에는 본인이 직접 `요청 취소` 가능

### 팀장 기준
1. 내 팀 화면에서 `대기 중 요청` 확인
2. 지원자 역할, 소개, 요청 시각 검토
3. `승인` 시 `team_members`에 추가
4. `거절` 시 요청 상태만 변경
5. 정원 도달 시 모집 상태를 자동으로 `마감` 처리

## 4. 이번 구현 범위에서 현실적인 화면 요소
- 팀 카드: `지원하기` 버튼
- 내 팀 영역: `대기 중 요청` 개수 배지
- 팀장용 패널: 지원자 목록, 승인/거절 버튼
- 지원자용 패널: 내가 보낸 요청 상태 목록

- 팀장 메모와 거절 사유 템플릿
- 팀 초대 방식과 팀 참여 요청 방식 분리
